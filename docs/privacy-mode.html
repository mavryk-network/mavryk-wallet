Key Design Decisions

  • Privacy mode hides all financial amounts with •••• — visually clean and universally understood.
  • Toggle lives in MainBanner (home screen balance area) — an eye icon next to the total balance.
  • Icons available: app/icons/eye-open-bold.svg and app/icons/eye-closed-bold.svg.
  • State in UIStore (privacyMode: boolean, default false) — persisted to browser.storage.local via explicit partialize allowlist.
  • A single PrivacyAmount atom wraps any rendered value. When privacyMode=true, renders •••• instead of children. Avoids modifying Money.tsx globally.
Known limitation (accepted): Raw balance values are still passed as children props to PrivacyAmount and are visible in React DevTools. For a browser extension threat model (no remote DOM access), this is acceptable. Document but do not change architecture.
Scope: Home screen only for Phase 1 — MainBanner total + token list balances. Toggle tooltip must say "Hide balances on home screen". Send form and staking balances deferred to follow-on PR.

Sub-phases

Phase 1 — Store + Atom

File: src/lib/store/zustand/ui.store.ts

  1. Add privacyMode: boolean to the UIState interface.
  2. Add togglePrivacyMode: () => void to the UIActions interface. (Required — TS strict mode will error if omitted.)
  3. Add privacyMode: false to the initial state object.
  4. Add the action implementation:
tstogglePrivacyMode: () => set(s => ({ privacyMode: !s.privacyMode })),
Critical: Add privacyMode: state.privacyMode to the existing partialize return object (lines ~129–141). Without this, the field is never written to browser.storage.local and silently reverts to false on every reload. Verification step 3 (persistence) will fail if missed.

Add selector hook at the bottom with existing selectors:

tsexport const usePrivacyMode = () => useUIStore(s => s.privacyMode);

New file: src/app/atoms/PrivacyAmount.tsx

tsximport React, { FC } from 'react';
import { usePrivacyMode } from 'lib/store/zustand/ui.store';

interface Props {
  children: React.ReactNode;
  className?: string;
}

export const PrivacyAmount: FC<Props> = ({ children, className }) => {
  const privacyMode = usePrivacyMode();

  if (privacyMode) {
    return (
      <span className={className} aria-label="Hidden balance">
        ••••
      </span>
    );
  }

  return <span className={className}>{children}</span>;
};
Note on layout: The wrapping <span> may affect flex layout at some callsites. If the parent uses flex and the balance is a flex child, add className="contents" so the span is transparent to layout. Assess per callsite during Phase 3.

Export from src/app/atoms/index.ts alongside other atoms.

Phase 3 — Apply PrivacyAmount to Balance Display Points

Do NOT touch: BalanceFiat.tsx and BalanceGas.tsx — these are dead code not rendered anywhere.

Target: MainBanner/index.tsx

Wrap the local inline BalanceFiat component's <Money> output and the BalanceGas inline component's output with <PrivacyAmount>.

Target: src/app/pages/Home/OtherComponents/Tokens/components/ListItem.tsx

Enumerate all balance display points and wrap each:

Display Location in file Wrapper pattern
Main crypto balance <CryptoBalance> callsite <PrivacyAmount><CryptoBalance .../></PrivacyAmount>
Main fiat balance Inside <FiatBalance> render-prop callback {({ balance, symbol }) => <PrivacyAmount>...</PrivacyAmount>}
Delegated crypto balance <CryptoBalance> for delegated row Same as main crypto
Staked crypto balance <CryptoBalance> for staked row Same as main crypto
Important: FiatBalance uses a render-prop pattern. PrivacyAmount cannot wrap <FiatBalance> at the component boundary — it must go inside the render-prop callback. Wrap the rendered output JSX, not the <FiatBalance> element itself.
Balance.tsx: Do NOT wrap inside Balance.tsx itself — keep the mask opt-in at the callsite level, consistent with not modifying Money.tsx globally.
